你的密码安全吗？

发表于 2017-03-14 | 分类于 原理简介 | | 次阅读

1.加密与解密的军备竞赛

  “如果连续三次输入错误密码，系统将自动锁定”。这样的提示很常见，不过，一个普通的陌生人有多大的机会在三次之内猜中你的密码呢？答案高得令人吃惊。差不多有5%的中国人会选择“123456”、“12345678”、“88888888”当中的一个。可不要觉得“连续三次”的限制能起到多大的作用，真正的黑客决不会老老实实地呆在登录界面上。他们通常会先侵入系统，把整个密码数据库复制到自己的电脑上（这一步俗称“拖库”），然后用“常见密码表”甚至“密码字典”一一尝试。小型的“密码字典”一般包含了所有八位以下的数字和大小写字母组合，看似数量庞大，但如今的个人电脑足以胜任每秒百万次的密码测试，破解密码（特别是纯数字密码）消耗的时间并没有许多人想象得那么长。

  几乎没有人喜欢费时费力地想出一个又长又复杂的密码，再去记住它。聊天室和论坛的密码确实不值得我们费这个工夫，然而当越来越多与金钱和隐私有关的电子账号需要密码时，很多人便习惯性地沿用了论坛的简单密码，由此埋下了隐患。

  现在，许多网站和机构开始意识到这个问题，并且以矫枉过正的态度强迫每个用户去想出一个长达十位，包含大小写字母、数字和特殊符号的密码，让用户不得不修改自己的常用密码——可惜他们下次登录的时候往往又不记得上次是怎么修改的了。

  加密与解密是一场军备竞赛，无论情愿与否，所有人的密码都正在变得越来越长，越来越复杂。但正如战场上进攻的一方总是掌握主动权一样，在与入侵者的对阵中，普通用户永远是处于下风的。

2.怎样的密码才安全

  回答这个问题，首先要知道风险来自何方。

  风险大致有三类。第一类来自现实生活。例如，某人想窥探同事的隐私，他可能会尝试用同事的手机号或者姓名拼音登录他的邮箱账号；或是一个走运的窃贼不满足于钱包里的现金，还想试试用失主身份证上的数字组合取出其银行卡里的钱。只要密码不是过分简单，输错次数限制就足以挫败他们。

  第二类来自恶意网站。互联网上充斥着诸如“测测你是某某电视剧中的哪个角色”、“输入姓名，测明年运势”之类的网页小游戏，其中有一些还会借口“保存测试结果”，让你设置密码。它们的目的是收集你的密码，因为你在这个网站输入的密码很可能也用在别的网站上。此外，某些不靠谱的数据库也会带来同样的后果，如2015年的网易163邮箱大规模泄密，连锁导致了多起其他网站的大规模盗号事件。密码强度对防范此类危险毫无作用，就像再安全的防盗门也防不住拿到了钥匙的贼一样。应对的策略，最好是在不同网站使用不同账号，也不随便使用重要的密码。

  第三类风险针对特定的对象，如受到严密监视的罪犯，此时密码本身是什么也不重要，权力机关自有办法让密码持有者乖乖交出来。

  通过以上的分类，我们看到密码的安全性包括两方面，一是它自身是否足够强，另一个则是怎样保管和使用它。很大程度上，这两方面是相互关联的：喜爱简单数字密码的用户，往往会不分场合地使用相同或相似的简单密码。作为结论，在日常生活中，应当在重要场合尽量使用大小写字母、数字、符号混合并且长度在八位以上的密码。在只能使用数字密码的场合，应当避免使用生日、手机、身份证号等数字。原则上，每个账号的密码都应当不同，并且定期更新。

3.管理密码

  安全和易用是管理密码时永恒的矛盾。完全去实践上述安全密码的原则几乎是人力不可为的。很多时候人们连输入一个复杂的密码都嫌麻烦，更不用说定期去更新它了。幸好在这个通讯发达的时代，我们有许多方便好用的辅助工具。这里介绍两种降低管理密码难度的方法：两步验证和密码管理软件。

  两步验证：以苹果公司的两步验证为例，开启两步验证后，别人即使知道你的账号密码，也无法在他的手机上登录你的账号，除非先在你的手机上授权同意。

  密码管理软件：收费软件1password 和免费软件keepass都能在手机和电脑等不同平台上使用。软件自带生成随机密码、自动填写密码的功能。使用者只需记住一个打开软件的密码，其余的密码都可以交由软件记录管理。
